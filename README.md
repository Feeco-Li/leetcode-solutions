### 💡 为什么能缩短这么多时间？

1. **没有“降维打击”的挫败感**：
以前你卡住可能是在死磕“动态规划的硬核推导”或者“单调栈的奇葩应用”，一道题能折腾你一下午，还打击自信。砍掉这些之后，剩下的都是**所见即所得、逻辑极度顺畅**的基础题。
2. **模板通用性极高**：
比如二分搜索、双指针、二叉树遍历、DFS 岛屿问题，它们在 LeetCode 上都有**几乎一成不变的代码模板**。你只要理解并背下这 4 个核心模板，就能直接套用解出几十道不同的题目。
3. **中小厂更看重“无 Bug 一次过”**：
中小厂的面试官其实很务实。他们宁愿要一个**能快速、无 Bug 写出反转链表和二分查找**的候选人，也不想要一个能勉强说出动态规划思路却连指针怎么指都会写错的人。
